### New features
- **Trends tab** (`app/(tabs)/trends.tsx`) — 3rd tab with a custom bar chart (plain View rectangles, no library). Shows last 6 months of spending per category. Uses new `fetchMonthlyTotals(months)` DB function. Includes summary chips (totals per category), legend, and empty state. Registered in `app/(tabs)/_layout.tsx`.
- **CSV export** (`app/(tabs)/history.tsx`) — Share icon in History header. Exports the currently displayed transactions (respects active filter) as a CSV. Format: `Date,Time,Category,Subcategory,Note,Amount`.
  - **Native:** writes to cache directory with `expo-file-system` (legacy entry) then `expo-sharing.shareAsync`.
  - **Web:** downloads via a `Blob` + temporary anchor element (no native module).
  - Import fix: uses `expo-file-system/legacy` — the default entry deprecated/throws on `writeAsStringAsync`/`cacheDirectory` in SDK 54+.

### Web SQLite debugging (major)
Chrome/web threw a cascade of expo-sqlite errors (`createSyncAccessHandle`, `sqlite3_open_v2`, `no such table: app_settings`). Resolved with an architecture change — see [[web-inmemory-db]]:
- **Web now uses `:memory:` SQLite**, native keeps persistent `chronobudget.db`.
- **Migrations moved into `getDb()`** (`openAndMigrate`) so every query waits for a migrated schema — fixed a race that native was masking.
- **WAL only on native.**
- CSV export (`expo-file-system`/`expo-sharing`) guarded to native-only (they don't work on web); export button hidden on web.
