### Web SQLite debugging (major)
Chrome/web threw a cascade of expo-sqlite errors (`createSyncAccessHandle`, `sqlite3_open_v2`, `no such table: app_settings`). Resolved with an architecture change — see [[web-inmemory-db]]:
- **Web now uses `:memory:` SQLite**, native keeps persistent `chronobudget.db`.
- **Migrations moved into `getDb()`** (`openAndMigrate`) so every query waits for a migrated schema — fixed a race that native was masking.
- **WAL only on native.**
- CSV export (`expo-file-system`/`expo-sharing`) guarded to native-only (they don't work on web); export button hidden on web.

### UI fix — over-limit indicator (`components/BentoCard.tsx`)
- Percentage was clamped at 100% (`Math.min(amount / limit, 1)`), hiding overspend. Now the `%` label uses the real ratio (e.g. **136%**) while the bar fill stays capped at 100% width.
- Over-limit state forces the deep magenta `#FF2D78` bar + label and adds a small `alert-circle` + `OVER` pill (faint magenta bg). Calm default preserved when under limit.
- Verified on web: Needs $68 with a $50 limit → 136% + OVER badge, legible on the dark card; Wants/Savings (no limit) unchanged.
