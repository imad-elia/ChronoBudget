### Bug — onboarding Continue button invisible on iOS (`components/OnboardingOverlay.tsx`)
Three-part fix across the session:
1. **Height overflow**: country card had no height cap; now capped via `useWindowDimensions().height - 80` (more reliable than `%` inside a `Modal`).
2. **Footer-pinned layout**: everything above the Continue button (icon, title, subtitle, country panel, hint) sits in a shrinkable `ScrollView`; the button is pinned outside it, so it stays visible on any screen height.
3. **Root cause of the invisible button — native-only flex collapse**: the button reused `styles.nextBtn`, which has `flex: 2` (meant for the tour card's *horizontal* actions row). In the country card's vertical layout, `flex: 2` implies `flexBasis: 0` → button collapses to **0 height**. Overriding with `flexBasis: 'auto'` fixed web but NOT native: **Yoga treats `flexBasis: 'auto'` as unset**, so `flex`'s implicit basis 0 still won on iOS. Final fix: the country Continue button uses its own fully self-contained `countryStyles.continueBtn` (fixed height 46, no `flex` property at all). ⚠️ Lesson: never "override" a `flex` shorthand on native — omit it entirely.
